SEO初學者教學，搞懂 SEO 讓你網站訂單接不完

如果你是第一次接觸 SEO，請先記住一件事：SEO 的目標不是討好搜尋引擎，而是讓使用者更快找到真正有用的答案。本教學會用最少的行話，帶你從「搜尋如何運作」一路走到「關鍵字研究、內容撰寫、技術優化與站外權威」，最後用一步步的案例把流程走完。做完這些，你不只會看懂報表，更會知道下一步要做什麼，並把它做對。
（參考：Google — How Search works）

1. 定義與重要性

前導段

SEO（Search Engine Optimization，搜尋引擎優化）是一組方法與流程，讓你的網頁更容易被 抓取（Crawl）、收錄（Index）、並在 排名（Rank） 時有更好表現。對商業來說，SEO 是最具「長期複利」的行銷投資：文章持續帶來自然流量、轉換與品牌信任。

A. 什麼是 SEO

  • 不是廣告、不是快速秘技：搜尋引擎不接受付費換較高排名；即使遵循規範，也不保證一定會收錄或顯示。重點在於持續輸出對人有用、對站點可被理解的內容與結構。 (Google SEO Starter Guide)
  • 目標：在對的關鍵字下，讓對的人在對的時候找到你，並在頁面上完成你要的行動（例如詢價、下單、預約）。

B. 為什麼重要

  • 可預測、可累積：不同於廣告拉掉預算就歸零，SEO 的內容與權威會累積。
  • 高意圖流量：用戶在搜尋框裡輸入的每個詞，都是需求與痛點的線索。
  • 降低整體獲客成本（CAC）：當自然流量占比提高，同樣預算下的營收槓桿更大。

2. 原理與流程

A. 抓取（Crawl）：找到頁面

  • Googlebot 會循著網頁連結與你提供的站點地圖（Sitemap）來探索新內容。
  • 你可用 robots.txt 指示「哪些路徑可抓、哪些不抓」，但它不是拿來「阻止收錄」的工具（要阻止收錄請用 noindex 或權限保護）。 (robots.txt 指南)
  • 站點地圖可加速新頁面被發現，並標示更新時間/多語版本等訊息。 (Sitemaps 概述)

B. 索引（Index）：理解與歸檔

搜尋引擎會解析你的 HTML、JavaScript、結構化資料（Schema），決定頁面內容主題與可顯示的強化結果（Rich Results）。重複或相似內容應指定正規網址（Canonical），避免權重分散與索引混亂。 (Structured Data 介紹)

C. 排名（Rank）：決定誰在前

排名會考慮內容的幫助性/可靠性、連結、使用體驗等多重訊號。Google 強調：不接受付費換更高排名。核心網頁生命力（Core Web Vitals）中的 INP 已在 2024/3 取代 FID，代表互動回饋品質；可透過 Search Console 與 PageSpeed Insights 量測與改善。 (INP 介紹)

3. 三大面向

3.1 技術

關鍵目標：讓搜尋引擎抓得到、看得懂、載得快、不會誤會。

  • 可抓取：良好的內部連結、乾淨的 robots.txt、提供 Sitemap。 (robots.txt)
  • 可索引/可理解：正確使用 title/meta description/alt/heading；以 Schema 提示內容語意與可出現的豐富結果。 (Schema 說明)
  • 正規化：重複內容指定 rel="canonical"；避免參數頁/排序頁被錯誤收錄。 (Canonical 指南)
  • 行動優先 & 響應式：以響應式為首選做法；確保行動版內容與桌機一致。 (Mobile-first 指南)
  • 速度 & 體驗：關注 LCP/CLS/INP，使用 PageSpeed Insights 與 Lighthouse 找瓶頸並持續監測。 (PageSpeed Insights)
  • 安全：全站 HTTPS（雖是輕量訊號，但屬良好實務）。

延伸閱讀：Google《Search Essentials》

3.2 內容

核心信念：為人寫、而不是為機器堆字。

  • 理解使用者意圖：分辨「資訊型 / 導航型 / 交易型」；同一關鍵字可有多種意圖，內容需對準。
  • 主題聚焦與結構：一頁一主題；H2/H3 條理清楚，摘要與重點列表降低跳出。
  • 可執行價值：示例、步驟、檢核表、下載資源。
  • 原創性與信任：作者署名、來源引用、清楚的聯絡/公司資訊；展現 E-E-A-T 的經驗與專業。 (評測員指南)
  • 關鍵字自然佈局：以讀者會搜尋的語彙命名標題與段落，而不是堆疊字詞。
  • 更新維護：過時內容（價格、規格、流程）要修正，避免誤導。

3.3 站外

站外訊號 = 別人對你的信任

  • 高品質連結：來自主題相關、可信網站的自然引用。
  • 合規的出站連結標記：付費或贊助內容用 rel="sponsored"；UGC 用 rel="ugc"；其他需要時可用 nofollow（現作為「提示」使用）。 (出站連結標記)
  • 避免違規手法：連結方案、隱藏關鍵字、橋頁、刮站等都屬垃圾策略，可能被降權或移除。 (Spam Policies)

4. 入門步驟

4.1 關鍵字研究

目標：找出「有量、有意圖、你能打的贏」的詞。

  1. 蒐集詞庫：從產品/服務 → 客戶常問 → 競品標題 → 搜尋建議/相關搜尋 → Google Trends。
  2. 判斷意圖：打開 SERP 看前 10 名類型（內容/分類頁/品牌頁），你的頁型要對齊。
  3. 分群與層級：將近義詞合併、一頁一主題、餘詞安排為子段落或 FAQ。
  4. 在地/多語：多語網站規劃 hreflang，避免錯送語言版本。
  5. 量測與決策：用 Google Ads 關鍵字規劃工具估量搜尋量與趨勢，決定優先級。

工具連結：Google Trends、關鍵字規劃工具

4.2 內容撰寫與頁面優化

寫作清單

  • 標題（Title）：自然放入主詞 + 具體承諾（約 35–55 字元視像素）。
  • 主標/H1：對齊標題但以人話說清楚。
  • 前導段：2–4 句把讀者要的答案講白。
  • 內文結構：H2/H3 條列 + 圖例/步驟，避免巨牆文字。
  • 內部連結：指向「下一步」與「相關主題」，幫助抓取與理解。
  • Meta Description：寫給人看的摘要，提升點擊率（非直接排名因子）。
  • 媒體替代文字（alt）：以「圖像扮演的資訊角色」來描述。
  • FAQ/結語/CTA：幫讀者收尾、給明確行動（諮詢/試用/下載）。

頁面加值：結構化資料（JSON-LD）可幫助爭取豐富結果資格（文章/FAQ/產品等）。

4.3 技術面優化

站點可抓取/可索引檢查

  • Search Console：用「URL 檢查」看索引狀態、可否抓取、行動相容性、結構化資料。
  • Sitemap：提交並監控處理狀態；也可在 robots.txt 宣告路徑。
  • Canonical 與重複內容：明確指定，避免參數 URL 分流權重。

速度與體驗

  • PageSpeed Insights / Lighthouse：檢查 LCP/CLS/INP，逐一修正圖片、字型、第三方程式與主執行緒阻塞。
  • 行動優先：以響應式為首選；確保行動端內容等同、可點擊區域足夠。
  • HTTPS：全站加密，避免瀏覽器「不安全」警示與信任流失。

4.4 站外推廣 / 權威

自然權威的四條路

  1. 可被引用的內容資產：原創研究、數據圖表、模板（最容易換到自然連結）。
  2. 合作與露出：業界社群、研討會、專欄、Podcast。
  3. 產品/內容社證（Social Proof）：案例研究、客戶評價、媒體報導。
  4. 合規標記出站連結：贊助/廣告/UGC 分流標記，避免違規。

5. 常見錯誤與最佳實踐

5.1 常見錯誤

  • 把 robots.txt 當「擋收錄」：它只控制抓取，不保證不被索引；敏感頁請用 noindex 或權限保護。
  • 無站點地圖/站內架構鬆散：新頁沒被發現、孤兒頁過多。
  • 參數/篩選頁泛濫：未 canonical、未封裝，導致重複內容與抓取資源浪費。
  • 只顧字數不顧意圖：內容堆砌、答非所問，違反「以人為本」。
  • 頁面體驗差：CLS 大幅位移、INP 遲緩、行動端可點區太小。
  • 連結作弊/置換：參與連結方案、未標記贊助/UGC，踩垃圾政策。
  • 國際/多語錯配：沒有 hreflang 導致錯送語言/地區版本。
  • 忽略 Search Console 告警：錯失索引、結構化資料、手動處分通知。

5.2 最佳實踐

  • 以人為本：真問題→真解法→可執行步驟與證據。
  • 技術打底：可抓可索引、核心網頁生命力健康、HTTPS 安全。
  • 主題聚落（Topic Clusters）：一頁一主題 + 內部連結把相關內容串成網。
  • 量測與迭代：Search Console/PSI 定期檢視，依實際查詢與點擊不斷修文。

7. 案例流程

情境：在地咖啡店要推「手沖咖啡課程」。
目標：3 個月內，讓「手沖咖啡課程 台北/新手/價格」等關鍵字取得穩定曝光與表單詢問。

里程碑清單

  1. 關鍵字地圖（第 1 週）
    • 主詞：手沖咖啡課程（台北）／延伸：新手、器材、價格、時間。
    • SERP 觀測：前 10 名以「課程介紹頁／心得文／部落格教學」為主，決定我們做課程著陸頁 + 3 篇知識文。
  2. 內容產出（第 2–4 週）
    • 著陸頁結構：課程重點、師資、時間地點、費用、常見問題（FAQ Schema）、報名 CTA。
    • 三篇知識文：新手手沖必備器材清單、手沖常見 7 個錯誤、不同比較實測（含圖表）。
    • 內連策略：知識文導回著陸頁；著陸頁 FAQ 回鏈知識文詳解。
  3. 技術與體驗（第 3–5 週）
    • Lighthouse/PSI 優化圖片、字型載入、阻塞腳本，INP/LCP 合格；行動端可點區優化。
    • Sitemap 提交、URL 檢查確認可索引，FAQ Schema 綠燈。
  4. 站外與社證（第 4–8 週）
    • 與在地美食社群合作一篇體驗文（標記 sponsored）；蒐集學員心得與 Google 評論。
  5. 量測與迭代（第 5–12 週）
    • Search Console 看查詢字詞與點擊頁，補寫「價格/時段/地點」更直白的段落與 FAQ。

成果預期：第 6–8 週看到長尾詞曝光與點擊成長；第 10 週起主詞穩定進入前兩頁。表單詢問數隨著 FAQ 完備度、學員心得上線而提升。

結語

把 SEO 想成一條三線並行的產線：技術打底 → 內容產出 → 外部信任。你不需要一次把所有事做到滿分；你需要的是一個可重複的節奏：每週一個主題、每月一份稽核、每季一次大整理。當內容真正解決問題、網站載入順暢、且有他站願意引用時，搜尋引擎自然會把你推到更多人面前。從今天挑一個關鍵字，依照第 4 章的步驟做下去，你會在 Search Console 裡，看到曲線慢慢上揚。加油！
